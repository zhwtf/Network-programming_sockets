Setup for testing

All testing will be on the Unix machine: csil-cpu2.csil.sfu.ca.

Use SSH to open two windows to that machine: one for the sender and

one for the receiver.

Test cases

1. Code quality:                                                                /5  pts

Good structure, sufficient comments, no compilation errors/warnings, readme file.

2. GBN with small message and 0 loss rate:                   /15  pts

Send a small  (10 byte)  message over a reliable link.

Setup: sndBufSize = rcvBufSize = 10, lossRate = 0 at both sides.

3. GBN with long message and 0 loss rate:                     /15  pts

Send a long (45 byte) message, which should be divided into 5

segments (MSS =10), over a reliable link.

Setup: sndBufSize = rcvBufSize = 3, lossRate = 0 at both sides.

4. GBN with one-way losses:                                          /15  pts

Send 5 small (10 byte) messages back-to-back over a link with

40% loss rate, ACKs are not lost.

Setup: sndBufSize = rcvBufSize = 10 at both sides. 

 lossRate = 0.40 at sender side, lossRate = 0 at receiver side.

5. GBN with two-way losses:                                         /15  pts

Send 5 small (10 byte) messages back-to-back over a link with

80% loss rate, ACKs can be lost .

We also reduce the buffer size to one to test the synchronization.

Setup: sndBufSize = rcvBufSize = 1 at both sides. lossRate = 0.80 at both sides.          

6. Selective Repeat with 0 loss rate:                             /15  pts

Send 6 small (10 byte) messages back-to-back.

Setup: sndBufSize = rcvBufSize = 10 at both sides. lossRate = 0 at both sides.

7. Selective Repeat with two-way losses:                     /20  pts

Send 10 long (45 bytes each) messages back-to-back over a link

with 40% loss rate, ACKs can be lost.

Setup: sndBufSize = rcvBufSize = 6 at both sides. lossRate = 0 .40 at both sides.
